## H-MODEL-ROUTING-CHAT-PROSE-GATE-BLINDSPOT-1 (OPEN, filed 2026-07-17, priority: unset)

This is a structural design gap, not a bug in any single gate, and it is not closed by the existing Codex-side hook mirror (`~/.codex/hooks.json`), which gates Codex's own tool calls in Codex's own session and has no bridge into a Claude Code chat transcript or vice versa. No fix is designed yet; closing it would need a new mechanism that content-scans free assistant/user text for gate-result-shaped claims and independently re-verifies them against a real transcript before anyone acts on them.

## H-LOOP-PUBLISH-PII-GATE-SELF-TRIPPING-1 (CLOSED 2026-07-17, filed 2026-07-17, priority: unset -- Mission Control to set)
`~/Claude/loop`'s post-commit hook (`scripts/auto-publish-on-commit.sh`, installed 2026-07-04) auto-publishes every commit on main to `~/loop-public` -> `github.com/Eobodoechine/loop-engineering`, and has been failing closed on every commit since 2026-07-09 -- confirmed via 6 consecutive same-day commits on 2026-07-17 alone (`b243a52`, `15b09b4`, `f4dac47`, `9e18116`, `9114307`, `71c2f19`), all ending `publish FAILED` in `~/Claude/loop/.git/hooks/auto-publish.log`; independently confirmed via live GitHub API that the public repo's newest commit is still `2026-07-09T00:48:37Z`. Root cause: the PII/scrub gate is self-tripping -- two of the repo's own research dossiers (`research/pii_gate_tooling_false_positive_fix_dossier.md`, `research/drifting-branch-reconciliation-methodology-2026-07-16.md`) describe the gate's own detection patterns (`/Use` + `rs/`-prefix paths, `sk-ant` + `-api`/`sk-proj-` key-format strings) in prose, which then re-trip those same patterns on every subsequent publish attempt. Because the publish is all-or-nothing over the whole extracted tree, this blocks every commit's content, not just the offending dossiers -- this very backlog file was itself blocked as pure collateral on its first publish attempt, with nothing flaggable of its own. Fix needs either: excluding known-safe pattern-documentation files from the scan, restructuring those two dossiers to avoid literal example strings the gate's own regex matches, or making the gate diff-aware (scan only newly-published lines) rather than whole-tree. Until fixed, the public GitHub mirror stays stale on every future commit too, silently.
